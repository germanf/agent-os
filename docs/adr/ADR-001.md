# ADR-001: FastAPI + uvicorn as Web Server

**Status:** Accepted

**Date:** 2026-06-01

## Context

The platform needs a modern web server that serves both a REST API and an SPA (React) from the same process. Requirements include: strong typing with automatic validation, native asyncio support for SSE streaming, extensible middleware for authentication, and low overhead in both development and production.

## Decision

Use FastAPI as the web framework with uvicorn as the ASGI server. FastAPI provides:
- Automatic request/response validation via Pydantic v2.
- Interactive OpenAPI documentation (/docs) with zero configuration.
- Native async/await support for SSE and I/O-bound operations.
- Starlette-compatible middleware system.
- Static file mount for serving compiled frontend assets.

uvicorn runs on port 8765 (internal), behind nginx as a reverse proxy.

## Consequences

- Positive: rapid development, type-safe, automatic docs, mature ecosystem.
- Negative: tied to a specific Python ecosystem (not interchangeable with Express/Rails).
- Neutral: the catch-all SPA handler must be the last route (final @app.get).

## Alternatives Considered

| Alternative | Pros | Cons |
|-------------|------|------|
| Flask + Gunicorn | Lighter, more widely known | No automatic validation, no native OpenAPI, no native async |
| Express (Node.js) | Same language as frontend | Breaks Python backend stack consistency |
| Django REST | Built-in ORM, mature | Overkill for simple API, synchronous by default |

## Risks

None significant. FastAPI has massive adoption and long-term support.
