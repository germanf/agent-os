# ADR-009: Ponytail as Behavioral Modifier via Plugin

**Status:** Accepted

**Date:** 2026-06-20

## Context

OpenCode uses system prompts to define agent behavior. Modifying these prompts requires editing internal configuration files. A simple, non-intrusive way to inject custom behaviors (conciseness, format, tone) is needed without modifying OpenCode's code.

## Decision

Use **Ponytail** (npm plugin `@dietrichgebert/ponytail`) as a declarative behavioral modifier. Ponytail:
- Injects text into OpenCode's system prompt via plugin hook.
- Has zero runtime dependencies — pure text injection at prompt build time.
- Configured with one line in `opencode.json`: `"plugin": ["@dietrichgebert/ponytail"]`.

The plugin was declared in `opencode.json` during Phase 1. Custom prompts are defined in later phases as skill needs emerge.

## Consequences

- Positive: zero code, zero runtime dependencies, decoupled from OpenCode core, easy to enable/disable.
- Negative: limited to text injection (no programmatic transformation), depends on an external npm plugin.
- Neutral: only works with OpenCode; does not apply to Claude CLI (which uses `--append-system-prompt`).

## Alternatives Considered

| Alternative | Pros | Cons |
|-------------|------|------|
| OpenCode custom hooks | Native, programmable | Requires modifying OpenCode, more complex |
| Preset JSON template | No dependencies | Manual, no pipeline integration |
| OpenCode fork | Full control | Unsustainable maintenance |

## Risks

The plugin may break compatibility with future OpenCode versions. Since it's a declarative plugin, replacement is a one-line change in `opencode.json`.
