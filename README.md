# Agentic Software Boutique

> **🇺🇸 English** · [🇪🇸 Español](docs/es/README.md)

A generic, domain-agnostic platform for AI-assisted software development. Web dashboard, multi-agent orchestration, MCP tool ecosystem, pluggable AI backends — wire it to your own project, codebase, and infrastructure.

**One command to run:**

```bash
curl -fsSL https://raw.githubusercontent.com/germanf/agent-os/main/install.sh | bash
```

Sets up everything — clones repo, creates venv, builds frontend, generates credentials, prints your login URL. No sudo, no system deps.

**Or use Docker:**

```bash
docker compose up -d
```

---

## What is this?

A self-hosted control center for AI coding agents. You get:

- **Web dashboard** — health monitoring, job management, chat with multiple AI backends, file uploads
- **Multi-agent orchestration** — DAG-based task graphs, parallel/sequential execution, SSE streaming
- **Pluggable backends** — Claude Code, OpenCode, Codex, Kimi, or any OpenAI-compatible HTTP API
- **MCP ecosystem** — 4 built-in tool servers (Memory, Notes, Kanban, Workflows) with admin UI
- **Kanban workflow** — task tracking with approvals, cron scheduling, feedback polling
- **Observability** — health checks, structured JSON logging, alerts, OpenTelemetry tracing, token accounting
- **Sandbox isolation** — Docker containers for parallel agent execution
- **Production hardening** — rate limiting, HTTP Basic Auth, HSTS, SQLite WAL backups

## Quick Start

### Option 1: One-liner install (recommended)

```bash
curl -fsSL https://raw.githubusercontent.com/germanf/agent-os/main/install.sh | bash
```

Your credentials are printed at the end — open `http://localhost:8765` and log in.

### Option 2: Docker

```bash
docker compose up -d
```

### Option 3: Manual

```bash
git clone https://github.com/germanf/agent-os.git
cd agent-os
docker compose up -d
```

### Configure

1. **AI backend** — set `OPENAI_API_KEY` (and optionally `OPENAI_BASE_URL`, `OPENAI_MODEL`) to enable chat
2. **Auth** — `DASH_USER` and `DASH_PASS` env vars (auto-generated by install.sh)
3. **Optional** — `VAULT_DIR` path to an Obsidian vault for browsing notes

Open `http://localhost:8765`.

## Architecture

```
nginx (port 80 → 443 redirect, 443 → proxy_pass :8765)
│ VPN-only, client_max_body_size 55M
│
uvicorn (port 8765) — FastAPI app
│ AuthMiddleware · HSTS · Tracing · RateLimiting
│ Routes: agents, jobs, chat, orchestrator, mcp, ...
│ SPA catch-all @app.get("/{full_path:path}")
│
├── runner.py        — SSE job subprocess runner
├── chat_store.py    — SQLite persistence (chats, projects, messages)
└── frontend/dist/   — React SPA (Vite build)
```

### Key Modules

| Directory | Purpose |
|-----------|---------|
| `dashboard/` | FastAPI backend + React SPA |
| `dashboard/backends/` | Pluggable chat backends |
| `dashboard/agents/` | Agent capability implementations |
| `dashboard/orchestrator/` | Multi-agent DAG task execution |
| `dashboard/mcp/` | MCP tool/resource server system |
| `sandbox/` | Docker-based agent isolation |
| `specs/` | Role definitions and development workflow |
| `docs/` | Bilingual documentation (EN/ES) |

## Development

```bash
# Backend
uvicorn dashboard.main:app --port 8765 --reload
ruff check dashboard/

# Frontend
cd dashboard/frontend
pnpm run dev       # Vite dev server
pnpm run build     # tsc -b + vite build
pnpm run test      # vitest
pnpm run lint      # eslint

# Validation
bash scripts/validate-workflow.sh
```
